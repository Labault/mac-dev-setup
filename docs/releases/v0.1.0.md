# Version 0.1.0 release criteria

Version 0.1.0 is the first documented and reproducible release of the Mac Dev Setup project.

This release does not aim to automate an entire macOS installation. It establishes a stable foundation containing the tools, configurations, documentation, and validation workflows that have been manually reviewed and accepted.

## Repository quality

- [ ] The working tree is clean.
- [ ] All expected files are committed and pushed to `main`.
- [ ] `pre-commit run --all-files` completes successfully.
- [ ] Markdown files pass `markdownlint-cli2`.
- [ ] Documentation links pass `lychee`.
- [ ] Repository formatting passes `editorconfig-checker`.
- [ ] Gitleaks reports no detected secrets.

## Homebrew environment

- [ ] `brew bundle check --file=Brewfile` completes successfully.
- [ ] Every accepted Homebrew formula is declared in the `Brewfile`.
- [ ] Every accepted Homebrew Cask application is declared in the `Brewfile`.
- [ ] Rejected tools are not declared in the `Brewfile`.

## Shell environment

- [ ] The documented Zsh configuration loads successfully.
- [ ] Powerlevel10k loads successfully in a new terminal session.
- [ ] Antidote loads the expected plugins.
- [ ] Repository shell scripts pass ShellCheck when applicable.
- [ ] Installation, verification, and rollback instructions are documented.

## GitHub Actions

- [ ] At least one GitHub Actions workflow exists.
- [ ] All workflow files pass Actionlint.
- [ ] Actionlint is integrated into `pre-commit`.
- [ ] The main CI workflow completes successfully on GitHub.
- [ ] The main CI workflow can be executed locally with `act`.
- [ ] The limitations of local workflow execution are documented.

## Documentation

- [ ] The README accurately describes the project scope.
- [ ] Every accepted tool has sufficient installation and verification documentation.
- [ ] Documentation links from the README are valid.
- [ ] Unsupported or untested environments are clearly identified.
- [ ] Apple Silicon support has been manually validated.
- [ ] Intel Mac support is explicitly described as best effort and untested.

## Release

- [ ] The final commit follows the repository commit conventions.
- [ ] The final commit is pushed to `main`.
- [ ] The GitHub Actions workflow is green on `main`.
- [ ] The release is tagged `v0.1.0`.
- [ ] The tag is pushed to GitHub.
- [ ] A GitHub release is created from the `v0.1.0` tag.
