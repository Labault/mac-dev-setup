# Version 0.1.0 release criteria

Version 0.1.0 is the first documented and reproducible release of the MacDevSetup project.

This release does not aim to automate an entire macOS installation. It establishes a stable foundation containing the tools, configurations, documentation, and validation workflows that have been manually reviewed and accepted.

## Repository quality

- [x] The working tree is clean.
- [x] All expected files are committed and pushed to `main`.
- [x] `pre-commit run --all-files` completes successfully.
- [x] Markdown files pass `markdownlint-cli2`.
- [x] Documentation links pass `lychee`.
- [x] Repository formatting passes `editorconfig-checker`.
- [x] Gitleaks reports no detected secrets.

## Homebrew environment

- [x] `brew bundle check --file=Brewfile` completes successfully.
- [x] Every accepted Homebrew formula is declared in the `Brewfile`.
- [x] Every accepted Homebrew Cask application is declared in the `Brewfile`.
- [x] Rejected tools are not declared in the `Brewfile`.

## Shell environment

- [x] The documented Zsh configuration loads successfully.
- [x] Powerlevel10k loads successfully in a new terminal session.
- [x] Antidote loads the expected plugins.
- [x] Repository shell scripts pass ShellCheck when applicable.
- [x] Installation, verification, and rollback instructions are documented.

## GitHub Actions

- [x] At least one GitHub Actions workflow exists.
- [x] All workflow files pass Actionlint.
- [x] Actionlint is integrated into `pre-commit`.
- [x] The main CI workflow completes successfully on GitHub.
- [x] The main CI workflow can be executed locally with `act`.
- [x] The limitations of local workflow execution are documented.

## Documentation

- [x] The README accurately describes the project scope.
- [x] Every accepted tool has sufficient installation and verification documentation.
- [x] Documentation links from the README are valid.
- [x] Unsupported or untested environments are clearly identified.
- [x] Apple Silicon support has been manually validated.
- [x] Intel Mac support is explicitly described as best effort and untested.

## Release

- [x] The final commit follows the repository commit conventions.
- [x] The final commit is pushed to `main`.
- [x] The GitHub Actions workflow is green on `main`.
- [x] The release is tagged `v0.1.0`.
- [x] The tag is pushed to GitHub.
- [x] A GitHub release is created from the `v0.1.0` tag.
